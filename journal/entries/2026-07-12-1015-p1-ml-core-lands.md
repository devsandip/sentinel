# P1 ML core lands, and the fairness check flags on its own

Previous: none (first entry).

Started Sentinel today. The brief is an interview credibility artifact for an
SVP AI Product Management role at a bank. The point is not that agents can do
data science. The point is that agentic AI can be made governed and shippable
in a regulated bank. So the ML stays deliberately small and the governance
harness gets the real effort.

Two early calls. First, Streamlit instead of Next.js plus FastAPI. Python only,
one deploy target, roughly half the build effort, faster to a live link. Same
backend logic underneath. Second, start at P1 (the real ML) rather than P0
scaffolding ceremony, so there are genuine numbers to look at in the first hour.

P1 is done. Logistic regression on UCI German Credit. AUC 0.80. The nice
surprise is the fairness check. It flags on its own. Young applicants (<=25) get
flagged as risky at 0.333 versus 0.190 for the 41-60 band, a disparity ratio of
0.569, well under the four-fifths 0.8 rule. That means the Validator agent later
gets a real "disparity exceeds threshold, review this" event, not a staged one.
Sex, by contrast, passes at 0.82. Both real.

The protected attribute is excluded from the model features, and the code says
so out loud. That is the responsible-AI signal the model card will repeat.

Next is the harness: audit, RBAC, PII, guardrails, eval gate, cost.
